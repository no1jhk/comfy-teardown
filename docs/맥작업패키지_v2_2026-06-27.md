## 2. B — 다운로드 무결성 체크 ★근본원인 확정됨
**왜 (오늘 최종 통찰):** VAE가 117KB 깨진 껍데기였음(정상 1.35GB/348MB). **근본원인 = 큰 모델 다운로드 중에 ComfyUI Desktop을 여러 번 재부팅 → 다운로드가 중간에 끊겨 빈 파일로 저장.** 느린 회선(100Mbps 공유 + Parsec 동시)이 이를 악화. 형이 "받는 중인지 몰랐다"가 핵심.
**구현 스펙:**
- 모델 다운로드 항목마다 **예상 정상 용량** 표기
- 문구: `LTX23_video_vae_bf16.safetensors → models/vae · 정상 약 1.35GB (받은 뒤 용량 확인. 수 KB/MB로 작으면 다운로드가 깨진 것 → 삭제 후 재다운)`
- ★ 추가 경고: **"큰 모델 다운로드 중에는 ComfyUI/PC를 재부팅하지 마세요. 끊기면 빈 파일이 됩니다"**
- safetensors가 비정상적으로 작으면 JSONDecodeError 발생 노트
**🟩 명령:**
```
모델 다운로드 안내(Inventory/Solution 모델 항목, buildInstallScript)에 무결성 체크를 추가해줘. 각 모델에 예상 정상 용량을 표기하고 "받은 뒤 용량 확인 — 수 KB/MB로 작으면 깨진 것이니 삭제 후 재다운"을, 그리고 "큰 모델 다운로드 중에는 재부팅 금지(끊기면 빈 파일)"를 경고로 넣어. 용량 정보 없으면 "용량 확인 필요"로 두고 지어내지 마. safetensors가 비정상적으로 작으면 JSONDecodeError가 난다는 트러블슈팅 노트도. 커밋 말고 변경요약만.
```

## 3. ★신규 — 트러블슈팅 패턴 사전 (형 비전의 핵심 구현체)
**왜:** "작업하며 겪는 시행착오를 전부 데이터화해서, 쓸수록 해결능력이 좋아지게." 정적 모델 DB는 못 키우지만 **"증상→원인→해결" 패턴은 안 변함** → 이게 데이터화의 정답.
**데이터:** `troubleshooting_patterns.json` (오늘 5개 패턴 시드)
**구현 스펙:**
- 에러 로그 입력이 있으면, 로그를 패턴 사전과 매칭
- 매칭되면 "이건 X 문제입니다 → 이렇게 확인/해결" 자동 제시
- 새 패턴은 수동 추가(지금) → 나중에 "이거 맞았어" 버튼으로 반자동 적립
**🟩 명령:**
```
src/data/troubleshooting_patterns.json을 읽어서, 사용자가 붙여넣은 에러 로그를 이 패턴들과 매칭하는 기능을 추가해줘. 각 패턴은 match(로그에 포함될 키워드/정규식)와 cause, fix, check를 가짐. 매칭되면 진단 결과 상단에 "감지된 알려진 문제" 섹션으로 cause+fix를 보여줘. 여러 개 매칭되면 다 표시. 매칭 없으면 섹션 미표시(회귀 없음). 커밋 말고 변경요약만.
```
**수용기준:** LTX 에러 로그(JSONDecodeError) 붙이면 "파일 손상 → 용량 확인" 패턴이 뜨면 통과

## 5. ★신규 — 경로 재작성 (내 PC 기준으로)
**왜:** Findings가 `D:\ComfyUI_Shared\models`로 표기했는데 내 PC는 N드라이브. JSON에 박힌 옛 경로를 정직하게 읽은 것(=정확한 경고)이나, 내 경로로 바꿔주면 더 유용.
**🟩 명령:**
