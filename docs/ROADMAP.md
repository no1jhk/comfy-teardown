## 단기

- **로그 파일 경로(`** Log path:`) 형식 실측 확인 (확인 필요)**
  근거: "로그 파일로 대신하기" 안내(커밋 64570fc)가 `parseComfyLog`의 `** Log path: ...comfyui.log` 추출에 의존한다. 저장소 대표 실로그 픽스처(RTX3090 Windows standalone)에 이 라인이 없어, 최신 ComfyUI가 실제로 이 형식을 출력하는지·다수 사용자(수동 설치·구버전)에게 실제로 뜨는지 미확인. 최신 ComfyUI 실로그로 형식을 실측 확인한 뒤 유지/일반 힌트 전환을 판단한다(CLAUDE.md 규칙3 미확인 형식). code-auditor 첫 감사 [하·추정] 이월.

- **altUnsized 렌더 일원화 + regression 실경로 커버 (카탈로그 size 미상 항목 등재 시 필수 선행)**
  근거: 확정 대체(promoted)의 기대 용량이 카탈로그에 없으면 altHeld를 차단하고 altUnsized("정상 용량 정보가 아직 없어 크기 확인 불가")로 발화한다(커밋 738c733·감사 대응). 현재 모든 alt(alias 2.13GB·z-image bf16 12.3GB)에 size가 있어 altUnsized는 **실발화 0인 방어 전용**이라, 렌더가 altHeld처럼 6표면 일원화(verb·넣기·다운로드) 안 됐고(메시지만 특수) regression도 합성 plan만 커버한다. 향후 카탈로그에 **size 미상 alt_of/alias를 등재하는 시점**에, altUnsized 행 렌더 일원화(verb·넣기·다운로드 정리)와 실 buildModelPlan 경로 durable 커버를 **선행**해야 한다. code-auditor 재재감사 [중]×2(Q2·Q3b) 이월.

- **서버리스 프록시 (Vercel Functions · Anthropic 호출 서버 이전)**
  근거: 현재 AI 진단(v1.1)은 클라이언트에서 API 키를 못 쓴다(Vite `VITE_` 키는 빌드 번들에 평문 인라인 → 배포본 노출). 키·CORS를 서버 함수에서 처리해야 배포본에서 AI를 안전하게 켤 수 있다. (현행 `AI_KEY` 가드는 PROD 시 키를 비워 dev 전용 — 프록시가 이 제약의 정식 해소책.)

- **fixture ASCII 정상화**
  근거: `test/fixtures/`의 한글·유니코드 파일명이 CI 러너·타 플랫폼·셸에서 깨지거나 매칭 실패할 수 있다(현재 테스트는 `startsWith` 프리픽스 탐색으로 우회 중). ASCII 파일명으로 정규화하면 이식성·CI 안정성이 오른다.

- **Vite / esbuild 업그레이드**
  근거: esbuild가 JSX 구문 에러를 출력하고도 `exit 0`으로 넘겨 빌드 파손을 잠복시킨 사고 이력(HISTORY 5ec1a68). 최신 버전의 진단 정확도·보안 패치를 반영하고, 커밋 게이트(build 에러 부재 판정)의 신뢰도를 높인다.

- **GitHub Actions CI (build + smoke + e2e)**
  근거: 현재 `build`·`smoke`(3부)·`e2e`(15케이스)를 로컬 수동 실행에 의존한다. push마다 자동 게이트로 돌리면 회귀·TDZ·JSX 파손을 병합 전에 차단(런타임 크래시 사고 9fd2b07·356f5db 계열 예방).

- **노드 번호 병기 후속 정련 (작업2 감사 [하] 3건 · 실발화 낮음)**
  근거: 노드 위치 병기(커밋 9ba7f8d·70a35ca) code-auditor 감사에서 [하] 3건 이월.
  (1) `nodeLocStr` title 폴백이 `n.type`(class_type)이라 무제목 코어 노드의 캔버스 display_name(UNETLoader→"Load Diffusion Model")과 불일치 — #ID 배지가 실앵커라 방어되나 라벨 텍스트는 캔버스와 상이. 코어 노드 display_name 매핑 또는 무제목 시 라벨 생략("#130 노드") 판단.
  (2) 같은 파일을 3개+ 노드가 서로 다른 title로 참조하면 한 줄 라벨이 비대(90자+ 랩) — 현재 "복수면 전부 나열"(작업2 스펙)이라 유지 중. 3개 초과 시 "#11 외 N곳" 축약 판단.
  (3) scanOpen 유도 줄("보유 모델을 걸러내려면 폴더 대조를 추가하세요")이 상시 노출이라(작업1 스펙) 펼침·대조 완료 후에도 명령형 잔존(stale nudge). "상시 노출" 스펙 유지 vs 완료 시 서술형 전환 판단. #env-step 고아 앵커(참조 0) 정리 병행.

- **카탈로그 제안 내보내기 (학습 루프 반자동화)**
  근거: 실전에서 LLM 진단으로 해결된 항목의 검증→등재가 수동 지시문 경유로 확인됨(SeedVR2 6종·Krea2 Edit 계열). learned cache 후보를 검증 필드(파일명·용량·직링크·근거)를 채운 JSON 스니펫으로 내보내 → 사용자 승인 → `model_catalog` 병합. 자동 등재는 금지 유지(날조 방지), 승인 1클릭까지만 자동화.
