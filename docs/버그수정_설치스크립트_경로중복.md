# Teardown — 설치 스크립트 폴더 경로 중복 버그 수정

> VS Code Claude Code용. 대상: src/Teardown.jsx, buildInstallScript() (약 724~734줄)
> ⚠ 작은 버그 수정. 실제 PC 검증 전에 꼭 고쳐야 함.

---

## 버그
스크립트 모델 경로가 `models/models/checkpoints`로 **models 중복**.
- 원인: `m.folder`(또는 info.folder)가 이미 `models/checkpoints` 형태로 저장됨(233줄 `models/${ci.folder}`, guessFolder도 `models/`로 시작).
- 그런데 buildInstallScript에서 `models/${folder}`로 **또 models/ 접두**를 붙임 → 중복.

## 수정 (약 728~734줄)
`models/` 접두사를 제거. folder 변수를 그대로 사용 (이미 models/ 포함).

현재:
```js
const folder = (info && info.folder) || m.folder || "models";
const fname = m.file.replace(/\\/g, "/").split("/").pop();
if (info && info.exact && info.url) {
  L.push(isWin ? `if not exist "models\\${folder}" mkdir "models\\${folder}"` : `mkdir -p "models/${folder}"`);
  L.push(isWin
    ? `curl -L -o "models\\${folder}\\${fname}" "${info.url}"`
    : `curl -L -o "models/${folder}/${fname}" "${info.url}"`);
} else {
  L.push(`${cmt} [수동] ${fname} → models/${folder} (URL 미확인. "이 모델 검색"으로 찾으세요)`);
}
```

수정 후 (folder가 이미 models/ 포함하므로 접두사 제거 + Windows는 역슬래시 변환):
```js
const folder = (info && info.folder) || m.folder || "models";  // 이미 "models/checkpoints" 형태
const folderWin = folder.replace(/\//g, "\\");                  // Windows용 역슬래시
const fname = m.file.replace(/\\/g, "/").split("/").pop();
if (info && info.exact && info.url) {
  L.push(isWin ? `if not exist "${folderWin}" mkdir "${folderWin}"` : `mkdir -p "${folder}"`);
  L.push(isWin
    ? `curl -L -o "${folderWin}\\${fname}" "${info.url}"`
    : `curl -L -o "${folder}/${fname}" "${info.url}"`);
} else {
  L.push(`${cmt} [수동] ${fname} → ${folder} (URL 미확인. "이 모델 검색"으로 찾으세요)`);
}
```

## 확인
- LTX JSON으로 .bat 생성 시 경로가 `models\checkpoints\ltx-2.3-22b-dev.safetensors` (models 한 번만).
- .sh도 `models/checkpoints/...` (중복 없음).
- 미확인 주석도 `→ models/checkpoints` (중복 없음).

## 완료 기준
1. 모델 경로에 models 중복 없음 (.sh/.bat 둘 다).
2. Windows는 역슬래시(\), Mac/Linux는 슬래시(/).
3. npm run build 에러 없음.
