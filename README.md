# Teardown

> 무거운 ComfyUI 파이프라인(Trellis2 · UniRig · HYMotion)의 워크플로 JSON을 해체해
> 필요한 노드 · 모델 · 패키지와 버전 충돌 · 이식 위험을 진단하는 도구.
> 설치는 사람이, 진단과 안내는 도구가.

## 실행

```bash
npm install
npm run dev      # → http://localhost:5173
```

빌드:

```bash
npm run build
npm run preview
```

## 무엇을 하나

워크플로 JSON 하나를 올리면(또는 "샘플로 보기"):

- **패키지 · 버전** — cnr_id별 pack과 버전. 한 pack에 버전이 여러 개면 *버전 충돌 흔적*으로 표시.
  같은 저장소에서 나오는 복수 cnr_id는 "한 번만 설치"로 통합 안내.
- **출처 추정 노드** — cnr_id 없는 노드를 이름 기준으로 저장소 추정. 프론트엔드 전용 노드는 제외.
- **이식 위험 값** — 경로 구분자 `\`, `flash_attn` 설정, 과거 산출물 경로 등 이식 시 깨지는 값.
- **모델 · 자산 인벤토리** — 참조 모델 파일 + 폴더 추정 + 파일명 리네임 힌트.
- **비활성 노드** — muted / bypass된 노드.

## 포지셔닝

일반 미씽노드 해결은 ComfyUI-Manager가 한다. 이 도구는 Manager가 손대지 못하는
**컴파일 의존성 지옥(flash_attn 등)**을 가진 무거운 파이프라인 진단에 집중한다.
