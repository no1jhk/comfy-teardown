---

빌드가 깨끗해질 때까지 자동으로 디버깅 루프를 돈다.

## 루프 절차

다음을 **에러가 0이 될 때까지 반복**한다 (최대 5회):

1. `npm run build` 실행
2. 에러/경고가 있으면:
   - 에러 메시지를 정확히 읽고, 원인 파일·줄을 찾는다
   - 근본 원인을 고친다 (증상만 가리지 말 것)
   - 다시 1번으로
3. 에러가 0이면 루프 종료

## 규칙 (중요)

- **추측으로 고치지 않는다.** 에러 메시지가 가리키는 실제 파일·줄을 읽고 원인을 확인한 뒤 수정.
- **회귀 금지.** 에러를 없애려고 기능을 삭제하거나 주석 처리하지 않는다. 진짜 원인을 고친다.
- **5회 안에 못 끝내면 멈춘다.** 무한 루프 방지. 5회째에도 같은 에러가 남으면, 루프를 멈추고 "이건 자동으로 못 고침 — 원인과 막힌 지점"을 사용자에게 설명한다.
- **같은 수정을 두 번 시도하지 않는다.** 한 번 시도해서 안 된 방법은 다시 안 쓴다 (다른 접근).
- lint 에러도 있으면 같이 본다 (`npm run lint`가 있으면).

## 마무리 보고

루프가 끝나면 사용자에게 보고:
- 성공 시: "빌드 통과 — N회 만에 해결. 고친 것: (요약)"
- 실패 시 (5회 초과): "자동 수정 한계 — 남은 에러: (에러 요약) / 원인 추정: (설명) / 사람 판단 필요한 부분: (무엇)"

커밋은 하지 않는다 (사용자가 확인 후 /end 로 커밋).
