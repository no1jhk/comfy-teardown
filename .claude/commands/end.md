---

이 세션을 완전히 마무리한다. 두 가지를 순서대로 실행:

## 1단계 — 작업 기록을 docs/HISTORY.md 맨 위에 추가

이번 세션에서 실제로 한 작업을 아래 형식으로 `docs/HISTORY.md` **맨 위**에 추가 (기존 내용은 아래로 밀기):

```
## YYYY-MM-DD HH:MM (세션 요약 제목)

**한 일**
- (이번 세션에서 실제로 수정/생성한 것. 파일명 포함)

**수정한 파일**
- 파일경로 — 무엇을 바꿨는지 한 줄

**현재 상태**
- (지금 어디까지 됐는지. v2 작업계획 5단계 중 몇 단계인지)

**다음 할 일**
- (다음 세션에서 이어서 할 것)

**주의/메모**
- (막힌 점, 검증 안 된 것 등. 없으면 생략)

---
```

규칙: 날짜·시간은 실제 현재 시각. 한국어, 개조식, 간결하게. 실제로 한 것만.

## 2단계 — git 커밋

기록을 남긴 뒤, 이번 세션의 모든 변경사항을 git 커밋한다:
- `git add -A` 로 변경사항 스테이징
- 커밋 메시지는 이번 세션 작업을 한 줄로 요약해서 작성 (예: "v2 1단계: compatibility.json 데이터층 연동")
- 커밋 메시지 끝에 `Co-Authored-By: Claude` 한 줄 포함
- push는 하지 않음 (로컬 커밋만)

## 마무리
두 단계 끝나면 사용자에게 이렇게 한 줄로 보고:
"세션 종료 — HISTORY.md 기록 완료 + 커밋됨([커밋메시지]). 다음 세션에서 HISTORY.md 읽으면 이어서 진행 가능."
